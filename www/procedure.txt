
Procedure to update the Change Log web page on new release.

0.  In directory .. = ~/proj/maketool
1.  Checkin all changes.
2.  Choose a release number, e.g. 0.5
3.  Tag the source:
    cvs tag MAKETOOL_0_5
4.  Build a list of change from the previous release (e.g. 0.4)
    www/cvs2changes 0.4 0.5
5.  Manually edit the CHANGES file to remove, re-express and
    re-order the change log items in the first section (just
    created by the cvs2changes script). Maintain the formatting.
6.  Checkin and re-tag CHANGES
    cvs ci CHANGES
    cvs tag -d MAKETOOL_0_5 CHANGES
    cvs tag MAKETOOL_0_5 CHANGES
7.  Rebuild the changelog.html page
    make -C www
8.  Connect to alphalink
9.  Install the web pages on alphalink
    make -C www install
    make -C www upload
